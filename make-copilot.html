Pourquoi Make pour cette brique ?

🧩 Le bon outil au bon endroit

Le reste du site est codé à la main sur Cloudflare Workers. Cette brique est volontairement en no-code pour montrer la maîtrise de l'orchestration visuelle : webhook, appel HTTP authentifié, réponse synchrone.

🔐 Clé jamais exposée

La clé Groq vit côté serveur dans le module HTTP de Make, jamais dans le navigateur. La page n'appelle que le webhook public, qui renvoie le résultat avec un en-tête CORS.

⚡ Synchrone via Webhook Response

Le module Webhook Response renvoie le JSON dans la même requête HTTP, ce qui permet un affichage instantané sans polling. L'appel part en x-www-form-urlencoded pour éviter le preflight CORS.

🪶 Léger par design

3 modules, environ 3 opérations par exécution. Un throttle côté client espace les requêtes pour préserver le quota du plan gratuit Make (1000 opérations par mois).
