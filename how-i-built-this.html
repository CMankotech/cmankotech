How I built this

Comment j’ai construit ce site et ses outils IA

J’ai construit ce portfolio et les outils PM avec une approche simple : des interfaces statiques rapides à charger, une couche proxy sécurisée pour les appels IA, puis une orchestration plus intelligente pour guider les réponses.

Architecture globale

1) Frontend statique

Chaque outil est une page HTML/JS autonome, déployée sur GitHub Pages. Aucun secret n’est exposé dans le navigateur.

2) Proxy Cloudflare

Les appels passent par un Worker (/orchestrate) qui centralise la sécurité, le routage et les fallbacks.

3) Orchestrateur LangGraph

Un service FastAPI + LangGraph produit un plan structuré puis une réponse concise via un flux planner → synthesis.

Ce qu’on a ajouté récemment (LangGraph)

  • Une route Worker POST /orchestrate qui tente d’abord l’orchestrateur externe quand il est disponible.
  • Un fallback natif vers Groq côté Worker si l’orchestrateur n’est pas joignable.
  • Un parsing plus robuste des sorties planner (y compris JSON entouré de blocs ```json).
  • Des tests automatiques sur l’orchestrateur et le proxy pour sécuriser les cas de routing/fallback.

Pourquoi ce choix ?

L’objectif est d’avoir une expérience fiable et actionnable : plan PM structuré + réponse claire, tout en gardant un site rapide, keyless côté utilisateur, et simple à maintenir.
